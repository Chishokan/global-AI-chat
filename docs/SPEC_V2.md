# 開発仕様書 v2 — 日本生活支援アプリ（B2Bマルチテナント）

第2案（開発仕様書）を反映した、実装の正となる要件。現行 Phase 1（地域型・多言語チャット＋
スプレッドシートログ）を土台に、B2Bマルチテナントへ拡張する。

## 確定スタック（ユーザー合意）

- フロント/API: **Next.js (App Router) + React + TypeScript**
- DB/認証/ベクトル: **Supabase（PostgreSQL + RLS + Auth + pgvector）**
- デプロイ: **Vercel**（最短。将来AWSも可）
- 生成AI: **Anthropic Claude**（現行 `lib/claude.ts` を継続。AWS移行時は Bedrock も可）
- RAG: KnowledgeBases をベクトル化して検索 → プロンプト注入（埋め込みモデルは別途選定）

## 1. システム概要

- 目的: 在日外国人（留学生・労働者）の生活支援＋所属機関（学校/企業）の規則・マニュアル問い合わせ対応の効率化。
- ユーザー: エンドユーザー（外国人留学生・労働者）／管理者（学校教職員・企業の労務人事）。
- 形態: マルチテナント型SaaS（クラウド）一部ハイブリッド。

## 2. 情報セキュリティ方針（シンプル・セキュア）

- **A. テナント隔離（RLS）**: `tenant_id` が一致するユーザーのみ自組織データにアクセス可。SupabaseのRow-Level Securityで実装。
- **B.「リンク案内」による機密保護（重要方針）**: クラウド/AIに載せるのは **非機密の一般ルール・FAQ のみ**。機密情報（詳細な社内規定・個人情報を含む文書等）は **アプリ内にテキスト保存しない**。AIは「社内LANのファイルパス」や「アクセス制限付きURL」を**案内するに留める**。
  - 例: 「詳細は社内ネットワークから `\\192.168.1.100\Shared\マニュアル` を確認してください」
- **C. エスカレーション誘導**: AIが判断できない/機密で回答すべきでない質問は、適切な社内窓口（人事部 等）へ案内するようプロンプトで制御。

## 3. 主要機能要件

### A. 共通
- **多言語AIチャット（RAG連携）**: 母国語で質問 → 翻訳＋社内/学内データ検索(RAG)＋AI回答生成。回答は**日本語＋母国語を併記**（※現行で併記実装済み）。
- **認証（テナントベース）**: ログイン時に所属「学校/企業」テナントへ紐付け。

### B. 学校向け (Tenant Type: School)
- プロファイル管理（国籍・趣味・日本語レベル等の登録/更新）。
- 情報提供・Q&A: アルバイト規則（週28h等）・生活マナー／校則・時間割・テスト日程／進学先・学費・奨学金／文化・観光。
- **【新規】進学相談の一次サポート（教員ナレッジ化）**: 教員の進路指導ノウハウ・各校特徴をナレッジDB化。進学質問（面接対策・学校レベル等）に教員知見ベースの一次回答を24時間提供。

### C. 企業向け (Tenant Type: Company)
- 社内マニュアルQ&A（就業規則・業務ルール・安全衛生）。
- 社内スケジュール/年間カレンダー確認。
- 福利厚生の制度案内・利用手順。
- エスカレーション（人事・総務など窓口案内）。

## 4. データベース設計概念（RDB基本）

- **Tenants**: 組織情報（種別 school/company、認証・セキュリティポリシー設定を含む）
- **Users**: 利用者（`tenant_id` で紐付け、role）
- **KnowledgeBases**: AI検索用の独自知識（ベクトル化して保存、RLSで厳密なアクセス制御）

> 既存の地域KB（佐世保/西海）は「全テナント共通の非機密の生活情報土台」として残し、各テナント独自KBを上乗せする。

## 実装メモ（設計方針）

- データアクセスは **supabase-js＋RLS**（利用者JWTでクエリ→RLSが効く）。サーバー専用操作のみ service role。
- `tenant_id` は profiles（users拡張）に保持し、RLSポリシーで `auth.uid()` → tenant を判定。
- リンク案内は KnowledgeBases に「案内先（パス/URL）」を非機密エントリとして登録。システムプロンプトで「機密は本文回答せず案内に留める」を強制。
- RAG: pgvector。埋め込みモデルは選定（候補: OpenAI text-embedding-3 / Voyage 等。生成はClaude）。
