# 会話ログをスプレッドシートに保存する（GAS Web App）

チャットの1往復（質問と回答）を Google スプレッドシートに1行ずつ記録する機能です。
Google Apps Script（GAS）の Web App を経由するので、サービスアカウントの鍵は不要、
設定はトークン1つだけです。

> ⚠️ **プライバシー**: 氏名などの直接識別子は保存しません（匿名のセッションIDで扱います）。
> 会話ログのAI解析・職員共有は、利用規約や初回オンボーディングで本人に明示する前提です。
> スプレッドシートの共有範囲（閲覧できる人）は必要最小限に絞ってください。

記録される列: `timestamp / sessionId / region / lang / easyJp / category / question / answer`

> 既に旧版（`locale` 列など）でデプロイ済みの場合は、`gas/Logger.gs` を最新版に貼り替えて
> **新しいデプロイ**をしてください。新版はヘッダー名に合わせて書き込み、足りない列は自動追加します
> （`region` / `lang` が増えます）。

---

## 仕組み（概要）

```
ブラウザ → /api/chat（応答を返す）
                └─ after(): カテゴリ・言語を分類 → GAS Web App に POST → シートに1行追記
```

- ログは応答を返した**後**に非同期で書くので、利用者の待ち時間は増えません。
- 環境変数（URL・トークン）が未設定ならログは**自動でオフ**。チャットは普通に動きます。
- ログの書き込みに失敗してもチャットは壊れません（エラーはサーバーログにのみ出力）。

## セットアップ手順

### 1. ログ用スプレッドシートを用意

ログを貯めたいスプレッドシートを開きます（FAQの正本とは**別シート/別ファイル推奨**）。
新規でもOK。`Logs` という名前のシートはスクリプトが自動作成します。

### 2. Apps Script を作成

1. スプレッドシートのメニュー **拡張機能 → Apps Script** を開く。
2. エディタの中身を消し、リポジトリの **[`gas/Logger.gs`](../gas/Logger.gs)** の内容を全部貼り付けて保存。

### 3. トークン（合言葉）を設定

1. Apps Script 左の **プロジェクトの設定（歯車アイコン）** を開く。
2. **スクリプト プロパティ → プロパティを追加**。
   - プロパティ: `LOG_TOKEN`
   - 値: 長いランダム文字列（例: パスワード生成で32文字程度）
3. 保存。この値は後で Vercel にも同じものを設定します（**コピーしておく**）。

> ランダム文字列の例（ターミナル）: `openssl rand -hex 24`

### 4. ウェブアプリとしてデプロイ

1. Apps Script 右上 **デプロイ → 新しいデプロイ**。
2. 種類の選択（歯車）→ **ウェブアプリ**。
3. 設定:
   - 説明: 任意（例: `chat logger v1`）
   - **次のユーザーとして実行: 自分**
   - **アクセスできるユーザー: 全員**
4. **デプロイ** をクリック。初回は Google アカウントの**承認**を求められます（許可）。
5. 発行された **ウェブアプリのURL（`.../exec` で終わる）** をコピー。

> 動作確認: そのURLをブラウザで開いて `{"ok":true,...}` が表示されればOK。

### 5. Vercel に環境変数を設定

**Vercel → プロジェクト → Settings → Environment Variables** に追加（Production / Preview / Development すべてにチェック）:

| Name | Value |
| --- | --- |
| `SHEETS_WEBAPP_URL` | 手順4でコピーした `/exec` URL |
| `SHEETS_WEBAPP_TOKEN` | 手順3の `LOG_TOKEN` と**同じ値** |

（ローカルで試すときは `.env` にも同じ2つを書きます。）

### 6. 再デプロイ

環境変数は再デプロイで反映されます。**Deployments → 最新 → Redeploy**、
または何か push してください。

## 動作確認

1. 公開URLのチャットで何か質問して回答を受け取る。
2. スプレッドシートの `Logs` シートに行が増えていればOK。
   - `category`（生活/手続き/学習/その他）と `locale`（ja/en/vi…）が自動で入ります。
   - 数十秒たっても増えないときは下の「うまくいかないとき」を確認。

---

## うまくいかないとき

| 症状 | 対処 |
| --- | --- |
| 行が増えない | Vercel の env 2つが正しいか／再デプロイしたか確認。GAS URL は `/exec` で終わるか。 |
| `unauthorized` が返る | `SHEETS_WEBAPP_TOKEN` と GAS の `LOG_TOKEN` が一致しているか。 |
| ブラウザでURLを開くとログイン要求 | デプロイの「アクセスできるユーザー」が**全員**になっているか。 |
| スクリプトを直したのに反映されない | GAS は**新しいデプロイ**または「デプロイを管理→編集→バージョン:新規」で再デプロイが必要。 |
| カテゴリ/言語が空のことがある | 分類は補助的なもの。失敗時は空のまま記録され、チャットには影響しません。 |

---

## コスト・運用メモ

- ログONのときは、1回の会話につき分類用に軽量モデル（既定: Haiku）を**1回**追加で呼びます。
  コストを抑えたい/不要なときは、Vercel の `SHEETS_WEBAPP_URL` を外せばログ機能ごとオフにできます。
- 分類モデルは `ANTHROPIC_CLASSIFY_MODEL` で変更可能。
- 将来 Phase 2 で PostgreSQL ＋ 日次レポートに移行する場合も、この `Logs` シートを
  初期データとして取り込めます（列構成をそのまま流用できます）。
